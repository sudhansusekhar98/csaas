# Implementation Plan: CSAAS UX Reconstruction

**Branch**: `001-ux-reconstruction` | **Date**: 2026-05-16 | **Spec**: specs/001-ux-reconstruction/

**Input**: User's coal sample anti-adulteration prototype — React + TypeScript + Tailwind + Vite

---

## Summary

Reconstruct the navigation architecture and screen organization of the CSAAS prototype to align with the physical process flow: Consignment Receipt → Sample Collection & Sealing → Sample Preparation (Pulverisation) → Sample Division (Splitting) → Lab Receiving. The redesign preserves all existing components but reorganizes them, standardizes terminology, adds contextual wayfinding, and restructures the dashboard around a live pipeline view.

---

## UX Problem Analysis

### What's broken in the current prototype

| # | Issue | Impact |
|---|-------|--------|
| 1 | Navigation split (Main / Modules & Controls) doesn't reflect process stages | Operators lose context when navigating between operational steps |
| 2 | "Lab Receiving" is buried under secondary nav | Lab analysts can't quickly reach their primary screen |
| 3 | No process breadcrumb on operational screens | Users don't know where they are in the 12-step chain of custody |
| 4 | Dashboard shows generic KPIs, not a pipeline status | Supervisors can't see shift-level flow at a glance |
| 5 | Exception logging requires leaving the current screen | Operators lose their place and sample context when raising an NCR |
| 6 | Terminology is inconsistent across screens | Increases cognitive load; conflicts with IS 436 coal sampling standards |
| 7 | `ExtendedViewType` hack in types.ts (alert-config not in ViewType) | Technical debt; indicates unplanned feature addition |
| 8 | Splitting Station has no list/session view | No way to switch between active splitting sessions |
| 9 | Sample Tracking stepper items are not interactive | Steps cannot be clicked to navigate or reveal detail |
| 10 | Sidebar "System Health" card (Monitoring 84 Units) is always shown | Clutters sidebar; monitoring context doesn't belong in every view |

```
QUALITY & COMPLIANCE
  ⑤ Non-Conformance      [FileWarning icon]  — Raise and review NCRs
  ⑥ Audit History        [History icon]      — Immutable event log
  ⑦ Reports              [FileBarChart icon] — Report builder

ADMINISTRATION
  ⑧ Personnel            [Users icon]        — Staff, roles, biometric enrollment
  ⑨ System Health        [Activity icon]     — Equipment monitoring
  ⑩ Alerts               [BellRing icon]     — Notification rules

  [Dashboard accessible from system logo / home button]
```

## Phase 1 Deliverables

### 1. Navigation Restructure (App.tsx + Sidebar)

Extract sidebar into `Sidebar.tsx`. Implement three nav groups:
- **Operations** (4 items): ordered to match process flow
- **Quality & Compliance** (3 items): NCR, Audit, Reports  
- **Administration** (3 items): Personnel, System Health, Alerts

Remove the "Monitoring 84 Units" promo card from the sidebar bottom — it's noise.

### 2. Type System Cleanup (types.ts)

Merge `ExtendedViewType` into `ViewType`. Add `'non-conformance'` to replace `'exception-logging'`.

```typescript
export type ViewType =
  | 'dashboard'
  | 'consignments'
  | 'tracking'
  | 'division-station'    // was 'splitting-station'
  | 'lab-receiving'
  | 'non-conformance'     // was 'exception-logging'
  | 'audit'
  | 'report-builder'
  | 'personnel'
  | 'system-health'
  | 'alert-config';
```

### 3. Dashboard Redesign (DashboardView.tsx)

Replace 3-card generic stats with a **Pipeline Status Bar** showing live counts at each stage:

```
[Consignments: 8] → [In Prep: 14] → [Division: 3] → [Lab Queue: 6]
```

Below that: Active NCRs panel + Today's completed samples.

Remove "Recent Shipments" table from dashboard (it duplicates ConsignmentsView).

### 4. ProcessBreadcrumb Component (NEW)

A horizontal step indicator shown on all operational screens (Consignments, Tracking, Division, Lab Receiving). Highlights current step, shows completed steps in green.

```
[1. Consignment] → [2. Collection] → [3. Prep] → [4. Division] → [5. Lab]
```

Clicking a completed step navigates to that screen.

### 5. Terminology Cleanup

| Screen / Label         | Old                     | New                         |
|------------------------|-------------------------|-----------------------------|
| Nav item               | Splitting Station       | Division Station            |
| Nav item               | Exception Log           | Non-Conformance             |
| Page title             | Industrial Exception Ledger | Non-Conformance Report  |
| Page title             | Splitting Control Station | Sample Division Station   |
| Form label             | Exception Category      | Non-Conformance Category    |
| Button                 | Submit to Supervisor    | Submit NCR                  |
| Nav group              | Modules & Controls      | Quality & Compliance        |
| Dashboard title        | Preparation Overview    | Live Pipeline               |

### 6. Inline NCR Trigger

Add a small "Flag Issue" button (amber, `FileWarning` icon) to the header of each operational screen. Clicking opens a condensed NCR modal pre-filled with the current stage and any active sample ID visible on screen.

### 7. Division Station Session List

Before the split detail view, show a list of active splitting sessions (similar pattern to ConsignmentsView table). Selecting a session navigates into the current SplittingStationView detail.

## Key Design Principles (from research)

1. **Process-first navigation**: The sidebar reflects the physical journey of a sample, top to bottom.
2. **Contextual wayfinding**: Every operational screen shows a breadcrumb of where it sits in the 12-step chain.
3. **Exception capture in-context**: Operators never have to leave their screen to raise an NCR.
4. **Consistent terminology**: All labels match IS 436 / CIL operational vocabulary.
5. **Dashboard = pipeline health**: Not a report, not a summary — a live status of where samples are right now.
